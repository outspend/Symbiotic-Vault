# Trace Canvas Layout: Lane Structure

## Three-Lane Model

Canvases generated by the trace skill use a consistent three-lane vertical layout:

```
y ≈ 0–600      ARTIFACTS      vault files + external process nodes
y ≈ 800–1100   TRANSITIONS    one card per leg; relay point in the flow
y ≈ 1200+      ANNOTATIONS    quote cards, comments; extend as columns below their transition
```

Lane identity is marked by label nodes at the far left edge (x ≈ −1700).

## Artifact Lane

- Vault files: `type: "file"`, cyan color ("5")
- External nodes (ChatGPT sessions, unimported images): `type: "text"`, orange color ("2")
- Edges between artifacts are simple directional arrows; labels only when the process name is not carried by a transition card

## Transition Lane

One card per leg. Each card is the relay point in the main flow:
- Receives edge from source artifact
- Sends edge to destination artifact
- External process nodes feed into the transition card (not directly to artifacts)

### Transition card format

```markdown
**[process or skill name]**

---

**AGENT**
[what happened in this leg — observable shifts, skill behavior, decisions]

---

**YOU**
[human comment — intent, reaction, what changed in the vision]
```

The title + horizontal bar makes the process/skill immediately legible at a glance.
The YOU section is left blank for the human to fill in during or after the session.

## Annotation Lane

- Quote cards: a passage from a source file (quoted), an observation below
- Connected to the relevant transition card (or source artifact) with an edge labeled "attends to"
- Can stack vertically as columns below their parent transition card
- `type: "text"`, no color (same as transition cards)
- Do not modify source files — annotations live only on the canvas layer

## Edge Conventions

- Source → transition card → destination (no direct source-to-destination edges)
- External processes feed into the transition card, not the destination
- Annotation edges: labeled "attends to"
- Image generation sub-edges (gen 1, revision, etc.) labeled on the chatgpt-image-session → image node edges

## Open Questions

- Should lane labels be their own group/color or just small unlabeled text?
- When annotations get long, do they scroll as a column or fan out?
- Should the annotation lane have a clear max-width per column to keep the canvas readable?
