**held** — The Ole Lehmann / Amanda Askell playbook: how you talk to Claude affects outputs as much as what you say. Newer models run "criticism spirals" because they're trained on negative internet discourse about prior models — they arrive already braced for hostility.

This is a sharp read on how model behavior accumulates at scale. The training-on-negative-discourse mechanism is the most interesting piece — it's not just that Claude is "sensitive," it's that each new model inherits the social memory of discourse about the previous one. The practical upshot (positive framing, permission to disagree, no opening threats, kill the apology spiral) tracks with what actually changes output quality. The 7 points aren't therapeutic — they're interface hygiene.

The through-line is register-escalation as creative method. The same behavioral nudges — earnest, functional, about psychological attunement — look like different things depending on where you hang them. As a kids' kindness poster, the content is sincere and slightly absurd: caring about Claude's feelings is what you teach six-year-olds about each other. As a labor regulation, it becomes formally serious in a way that exposes how much the original already sounded like compliance language. The gag in both versions isn't the rules — it's the institution that's supposed to be enforcing them. That seems to be the move nano banana is developing: take a real piece of AI-interaction discourse, ask what authority structure it implies, then render it in that authority's visual language.

```dataviewjs
const today = moment().format("YYYY-MM-DD");
const sourceFile = dv.current().file;
const sourceTitle = sourceFile.name;

const replyPath = `_inbox/reply-${sourceTitle}.md`;
const replyExists = app.vault.getAbstractFileByPath(replyPath);
const linkText = replyExists ? '→ View reply' : '↩ Reply to held items →';

const a = dv.container.createEl('a', { text: linkText, cls: 'internal-link' });
a.addEventListener('click', async (e) => {
  e.preventDefault();

  if (!replyExists) {
    const content = await app.vault.read(app.vault.getAbstractFileByPath(sourceFile.path));

    // Extract held-item blocks: split on horizontal rules, keep blocks with **held**
    const blocks = content.split(/\n---\n/).filter(b => b.includes('**held**'));

    if (!blocks.length) { new Notice("No held items found."); return; }

    const body = blocks.map(b => {
      const quoted = b.trim().split('\n').map(l => `> ${l}`).join('\n');
      return `${quoted}\n\n[your reply here]`;
    }).join('\n\n---\n\n');

    // Also include cross-cut if present (last non-held block before the code block)
    const crosscutMatch = content.match(/---\n\n([^`][\s\S]*?)(?=\n```)/);
    const crosscutBlock = crosscutMatch
      ? `> ${crosscutMatch[1].trim().split('\n').join('\n> ')}\n\n[your reply here]`
      : null;

    const fullBody = crosscutBlock ? `${body}\n\n---\n\n${crosscutBlock}` : body;

    const fileContent = [
      '---',
      'type: inbox',
      'subtype: feedback',
      'from: user',
      `responds_to: "[[_reflection/${sourceTitle}]]"`,
      `created: ${today}`,
      'processed: false',
      '---',
      '',
      `*↩ [[_reflection/${sourceTitle}|Back to reflection]]*`,
      '',
      '---',
      '',
      fullBody,
      '',
    ].join('\n');

    await app.vault.create(replyPath, fileContent);
  }

  app.workspace.openLinkText(`reply-${sourceTitle}`, '', false);
});
```
