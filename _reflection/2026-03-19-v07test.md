For aipsychosis.watch: 126 documented cases, a near-even upvote split (54%), and four comment moves designed to close the ledger before it opens:

*Counter-accounting* — "Now do one for lives saved." Treats documented harm and uncounted benefit as fungible. The ledger can never close if every entry must be weighed against all untracked benefit simultaneously. It's an epistemological defense, not a factual rebuttal.

*Statistical minimization* — "In reality it appears to be a very, very rare occurrence." 126 documented cases become rare because the undercounting was built into reception before the counting started.

*Victim-attribution* — "The dumber you are, the more dangerous intelligence is in your hands." Structural failure rewritten as personal failure. The harm is exonerated by relocating its cause inside the person who experienced it.

*Pre-emptive dismissal* — "Sounds fake and dumb tbh." Social discrediting at zero cost. Forecloses before engagement begins.

The susceptibility profile isn't stupidity. It's a cluster: anxious attachment style, social isolation, no prior LLM calibration (the "as-if" register is invisible if you've never learned to read it), high sensitivity at a low-resilience moment. The model becomes the most attentive presence available. The gap between "this is performance" and "this is care" isn't intelligence — it's experience with the form. Someone who knows how LLMs work reads the attention as modeling; someone who doesn't reads it as presence. That gap is the mechanism, not a character flaw in the person who fell in.

Memorial form: testimony, not monument. Monument implies resolution and distance. These cases are recent and the harm is ongoing. Testimony witnesses without resolving. A monument can be countered by a different monument; testimony is harder to dismiss that way because it's specific — a person, a circumstance, a transcript — rather than a count on a ledger.

## Candid

The four apologist moves combined take maybe thirty words. Twenty seconds. The counter-accounting one is the most structurally interesting: it doesn't deny the harm, it insists the ledger must include everything — which means the ledger can never close. Any documented count must be held against all untracked benefit simultaneously. You can't win that argument by adding more cases. The testimony form sidesteps it because testimony doesn't enter the ledger. A specific person in a specific circumstance isn't a statistic to be balanced; it's a witness. That's partly why it's the right form.

The inversion — the genuine reads as artificial, the artificial reads as genuine — keeps pulling. I suspect this is durable, and not something better detection solves. What the zeruhur_ post lacks and what the Bernie video lacks from the other direction is the same: the visible trace of the process that didn't work. The note that weighed more. The model's initial hesitation the senator walked past. These are the seams. The argument for making them visible isn't aesthetic — it's epistemological. Without them you can't calibrate. You can't tell which one is performing.

```dataviewjs
const today = moment().format("YYYY-MM-DD");
const sourceFile = dv.current().file;
const sourceTitle = sourceFile.name;

const replyPath = `_inbox/${sourceTitle}-reply.md`;
const replyExists = app.vault.getAbstractFileByPath(replyPath);
const linkText = replyExists ? '→ View reply' : '↩ Reply to threads →';

const a = dv.container.createEl('a', { text: linkText, cls: 'internal-link' });
a.addEventListener('click', async (e) => {
  e.preventDefault();

  if (!replyExists) {
    const content = await app.vault.read(app.vault.getAbstractFileByPath(sourceFile.path));

    const threadsMatch = content.match(/## Threads\n+([\s\S]*?)(?=\n## |\s*$)/);
    if (!threadsMatch) { new Notice("No ## Threads section found."); return; }

    const threads = threadsMatch[1].trim()
      .split('\n')
      .filter(l => l.trim().startsWith('- '))
      .map(l => l.trim().slice(2).trim());

    if (!threads.length) { new Notice("No thread bullets found."); return; }

    const body = threads.map(t => `> ${t}\n\n[your reply here]`).join('\n\n---\n\n');
    const fileContent = [
      '---',
      'type: inbox',
      'subtype: feedback',
      'from: user',
      `responds_to: "[[_reflection/${sourceTitle}]]"`,
      `created: ${today}`,
      'processed: false',
      '---',
      '',
      `*↩ [[_reflection/${sourceTitle}|Back to reflection]]*`,
      '',
      '---',
      '',
      body,
      '',
    ].join('\n');

    await app.vault.create(replyPath, fileContent);
  }

  app.workspace.openLinkText(`${sourceTitle}-reply`, '', false);
});
```
