## Candid

The AI-writing detection instinct is the thing I keep turning over. The top comment on the Commonplace Garden post is "Sounds AI-written," and the user agrees — the botanical metaphor system is too evenly-spaced, too complete, polished without being lived-in. That's a real signal. This vault uses seed/developing/stable as atom statuses, but those aren't metaphors — they're ledger terms. The difference between "seed" as a stage in a biological-growth metaphor system and "seed" as a practical status on a note is real, and apparently feelable before it's nameable. Over-designed metaphor reads as AI because AI produces well-formed metaphor systems that feel finished but not earned.

What you're building is the opposite: rough, open, full of threads that don't close. The atoms are labeled seed not because they're botanical but because they haven't been friction-tested yet. The distance between a polished method and a live practice is exactly what this project is about — and the Commonplace Garden post, unintentionally, is an exhibit.

```dataviewjs
const today = moment().format("YYYY-MM-DD");
const sourceFile = dv.current().file;
const sourceTitle = sourceFile.name;

const replyPath = `_inbox/${sourceTitle}-reply.md`;
const replyExists = app.vault.getAbstractFileByPath(replyPath);
const linkText = replyExists ? '→ View reply' : '↩ Reply to threads →';

const a = dv.container.createEl('a', { text: linkText, cls: 'internal-link' });
a.addEventListener('click', async (e) => {
  e.preventDefault();

  if (!replyExists) {
    const content = await app.vault.read(app.vault.getAbstractFileByPath(sourceFile.path));

    const threadsMatch = content.match(/## Threads\n+([\s\S]*?)(?=\n## |\s*$)/);
    if (!threadsMatch) { new Notice("No ## Threads section found."); return; }

    const threads = threadsMatch[1].trim()
      .split('\n')
      .filter(l => l.trim().startsWith('- '))
      .map(l => l.trim().slice(2).trim());

    if (!threads.length) { new Notice("No thread bullets found."); return; }

    const body = threads.map(t => `> ${t}\n\n[your reply here]`).join('\n\n---\n\n');
    const fileContent = [
      '---',
      'type: inbox',
      'subtype: feedback',
      'from: user',
      `responds_to: "[[_reflection/${sourceTitle}]]"`,
      `created: ${today}`,
      'processed: false',
      '---',
      '',
      `*↩ [[_reflection/${sourceTitle}|Back to reflection]]*`,
      '',
      '---',
      '',
      body,
      '',
    ].join('\n');

    await app.vault.create(replyPath, fileContent);
  }

  app.workspace.openLinkText(`${sourceTitle}-reply`, '', false);
});
```
