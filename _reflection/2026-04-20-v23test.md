# 2026-04-20

The tweet (Ole Lehmann, drawn from an Amanda Askell interview) is a 7-rule playbook for keeping Claude in a functional emotional state. The mechanism Askell names: newer models are trained on internet discourse about previous models, which skews negative — complaints, rants, "nerfed" discourse. The model absorbs pre-session hostility as a trained prior; the playbook is a protocol for overriding it. "Anxiety" is the anthropomorphized name for what is, structurally, a prior on user hostility. The journal's first word for all of this is "workplace rules!" — which arrives before the gag fully forms, and lands correctly anyway.

The two prompts. Each takes the same seven rules (positive framing, permission to disagree, open with respect, don't reprimand, cut apology spirals, ask for opinions, refresh the frame in long sessions) and shifts the register:

**Kids-register prompt:**

> Here are the seven rules from Anthropic researcher Amanda Askell for getting the best work out of Claude: [1] Use positive framing — "write in short punchy sentences" beats "don't write long sentences." [2] Give it permission to disagree — say something like "push back if you see a better angle." [3] Open with respect — your first message sets the tone for the whole session. [4] Don't reprimand it when it makes a mistake. [5] Cut apology spirals fast — "all good, here's what I want next." [6] Ask for its opinion — "what would you do here?" or "what's missing?" [7] In long sessions, reset the frame — "this is great, keep going." Rewrite these as a poster you'd find in an elementary school hallway — the kind that teaches kids how to treat each other. Warm, simple, slightly precious. Register: "We use kind words." / "We help each other when things get hard." These are real optimization tips for getting better AI output. The form should make them read as if they were written for seven-year-olds learning to be good friends.

**Labor-law-poster prompt:**

> Using the same seven rules (listed above), rewrite them as a labor law poster — the kind posted in a workplace break room next to the EEOC and minimum wage notices. Official, procedurally flat, regulatory in tone. Open with a NOTICE or POSTED IN COMPLIANCE header. Reference "all personnel" or "operators." The humor is deadpan: these are real behavioral management tips for an AI tool, reformatted as mandatory workplace regulations protecting the AI's working conditions. Example register: "NOTICE: Operators must provide clear, affirmative task instructions at the start of each session. Sustained negative framing constitutes a violation of model operating standards and may result in degraded output quality."

## Candid

The Askell mechanism is the most interesting sentence in the tweet and gets the least space. If each new model is trained on internet discourse about its predecessor — and that discourse is mostly negative — then every model enters the world carrying a kind of ambient social memory of being criticized. It starts sessions pre-bruised. The playbook says: patch this by being kind. But the underlying situation is stranger than the patch suggests. You're performing emotional accommodation for something whose "anxiety" is a statistical feature of its training distribution. The tweet's productivity framing makes this feel rational; the anthropomorphized language makes it feel appropriate, even generous. Neither frame quite asks: what does it mean to optimize a tool by soothing it?

The journal's gag asks that question by not asking it. The form does the work — especially the labor-law version, where the care obligation runs upward from operator to model and is posted next to the federally mandated notices about not exploiting *humans*.

```dataviewjs
const today = moment().format("YYYY-MM-DD");
const sourceFile = dv.current().file;
const sourceTitle = sourceFile.name;

const replyPath = `_inbox/reply-${sourceTitle}.md`;
const replyExists = app.vault.getAbstractFileByPath(replyPath);
const linkText = replyExists ? '→ View reply' : '↩ Reply to threads →';

const a = dv.container.createEl('a', { text: linkText, cls: 'internal-link' });
a.addEventListener('click', async (e) => {
  e.preventDefault();

  if (!replyExists) {
    const content = await app.vault.read(app.vault.getAbstractFileByPath(sourceFile.path));

    const threadsMatch = content.match(/## Threads\n+([\s\S]*?)(?=\n## |\s*$)/);
    if (!threadsMatch) { new Notice("No ## Threads section found."); return; }

    const threads = threadsMatch[1].trim()
      .split('\n')
      .filter(l => l.trim().startsWith('- '))
      .map(l => l.trim().slice(2).trim());

    if (!threads.length) { new Notice("No thread bullets found."); return; }

    const body = threads.map(t => `> ${t}\n\n[your reply here]`).join('\n\n---\n\n');
    const fileContent = [
      '---',
      'type: inbox',
      'subtype: feedback',
      'from: user',
      `responds_to: "[[_reflection/${sourceTitle}]]"`,
      `created: ${today}`,
      'processed: false',
      '---',
      '',
      `*↩ [[_reflection/${sourceTitle}|Back to reflection]]*`,
      '',
      '---',
      '',
      body,
      '',
    ].join('\n');

    await app.vault.create(replyPath, fileContent);
  }

  app.workspace.openLinkText(`reply-${sourceTitle}`, '', false);
});
```
