A speculative method: use LLM assistance to articulate an entire game system — rules, affordances, interactions — as a structured knowledge artifact (likely a knowledge graph). Once externalized, the system becomes inspectable and simulatable: you can run QA by asking the system questions, surfacing inconsistencies, testing edge cases against "worries and insecurities."

From the inbox: "I think of game design and whether the right LLM assistance would enable me to articulate a whole game system. I figure this would be pretty impactful for QA — we could sim the game according to certain questions, worries, insecurities, etc."

Distinct value over manual documentation: LLM assistance lowers the barrier to full articulation; the KG representation makes the system inspectable and simulatable in ways prose documentation doesn't.
